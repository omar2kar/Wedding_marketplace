# Wedding Marketplace - نظام التقييمات والمراجعات

## المميزات المضافة

### 1. نظام التقييمات للعملاء
- التحقق من شراء الخدمة قبل السماح بإضافة تقييم
- عرض شارة "عميل حقيقي" للتقييمات المؤكدة
- نظام تقييم بالنجوم من 1-5
- إمكانية إضافة تعليق مفصل

### 2. لوحة إدارة التقييمات للبائعين
- عرض جميع التقييمات الخاصة بالبائع فقط
- إحصائيات شاملة (متوسط التقييم، توزيع النجوم، إجمالي التقييمات)
- فلترة التقييمات (مع رد/بدون رد)
- ترتيب التقييمات (حسب التاريخ أو التقييم)

### 3. نظام الردود على التقييمات
- إمكانية رد البائع على تقييمات العملاء
- تعديل أو حذف الردود
- عرض تاريخ الرد

## البنية التقنية

### Frontend (React)
- **VendorReviews.tsx**: مكون إدارة التقييمات للبائع
- **AddReview.tsx**: مكون إضافة التقييمات للعملاء
- **VendorDashboard.tsx**: دمج نظام التقييمات في لوحة التحكم

### Backend (Node.js + Express)
- **routes/reviews.ts**: جميع نقاط API للتقييمات
- **middleware/auth.ts**: التحقق من هوية العملاء
- **middleware/vendorAuth.ts**: التحقق من هوية البائعين
- **database.ts**: الاتصال بقاعدة البيانات MySQL

### قاعدة البيانات (MySQL)
- جدول `reviews` مع دعم الردود
- التحقق من الشراء عبر جدول `bookings`
- تحديث تلقائي لمتوسط تقييم البائع

## التشغيل

### متطلبات النظام
- Node.js v14+
- MySQL 5.7+
- XAMPP (للبيئة المحلية)

### إعداد قاعدة البيانات
```bash
cd backend
mysql -u root < database/schema.sql
```

### تشغيل Backend
```bash
cd backend
npm install
npm run dev
# الخادم يعمل على http://localhost:5000
```

### تشغيل Frontend
```bash
cd frontend
npm install
npm start
# التطبيق يعمل على http://localhost:3000
```

## نقاط API الرئيسية

### للعملاء
- `GET /api/services/:serviceId/purchase-status` - التحقق من حالة الشراء
- `POST /api/services/:serviceId/reviews` - إضافة تقييم جديد
- `GET /api/services/:serviceId/reviews` - عرض تقييمات خدمة

### للبائعين
- `GET /api/vendor/:vendorId/reviews` - عرض تقييمات البائع (خاص)
- `POST /api/vendor/reviews/:reviewId/reply` - إضافة رد
- `PUT /api/vendor/reviews/:reviewId/reply` - تعديل رد
- `DELETE /api/vendor/reviews/:reviewId/reply` - حذف رد

## الأمان والخصوصية
- التقييمات تظهر فقط للبائع المعني
- التحقق من JWT للمصادقة
- التحقق من الشراء الفعلي قبل السماح بالتقييم
- فصل صلاحيات العملاء عن البائعين
